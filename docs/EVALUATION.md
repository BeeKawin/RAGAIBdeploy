# Evaluation Guide

RAGAI-B has two main evaluation modes plus one live demo evaluation mode.

## Evaluation Metrics

Current answer metrics:

- `correctness`: factual and mathematical accuracy.
- `groundedness`: support from retrieved context.
- `completeness`: coverage of key points.
- `clarity`: clear explanation for high-school learners.
- `type_alignment`: match with selected answer style.
- `overall_band`: 1-10 score calculated directly from the five metric scores.

Supported answer types:

- `exam-prep`
- `concept-focused`
- `homework-help`
- `quick-answer`
- `general`

## Overall Band Formula

```python
overall_band = round(
    2.25 * (
        correctness * 0.35 +
        groundedness * 0.25 +
        completeness * 0.15 +
        clarity * 0.15 +
        type_alignment * 0.10
    ) - 1.25
)
```

Then clamp to `1-10` and apply score caps for low correctness, groundedness, completeness, or type alignment.

## Mode 1: Self-Made Evaluation Dataset

Example file:

```text
physics_bee_20.csv
```

The loader supports Bee-style headers:

- `answer` → `reference_answer`
- `type` → `preferred_answer_type`
- `keypoints` → stored and used as soft coverage hints
- `language` → passed into generation and evaluation

Run:

```powershell
.\.venv\Scripts\python.exe -m evaluation.run_eval --dataset .\physics_bee_20.csv
```

Outputs:

```text
data/eval/results/*.jsonl
data/eval/summaries/*.json
```

## Mode 2: Model-Generated Evaluation Set

Use this when questions and reference answers are generated by another strong model.

Expected fields:

- `id`
- `subject`
- `grade`
- `question`
- `reference_answer`
- optional `preferred_answer_type`
- optional `language`
- optional `keypoints`

The RAG answer is still judged against:

- the question,
- the reference answer,
- retrieved context,
- answer type.

## Mode 3: Live Query A/B Evaluation

Endpoint:

```text
POST /eval/live-query
```

Purpose:

- Test manually typed questions.
- Generate a RAG answer.
- Generate a baseline LLM answer without retrieved context.
- Evaluate both answers side by side.

This mode does not require a reference answer. It uses retrieved curriculum context as the grounding source.

Stored output:

```text
data/eval/live_queries/YYYYMMDD.jsonl
```

## Retrieval Evaluation

Retrieval evaluation checks whether the system retrieved chunks from the expected topic.

Run:

```powershell
.\.venv\Scripts\python.exe -m evaluation.retrieval_eval --dataset .\physics_bee_20.csv --top-k 6
```

Key fields:

- `retrieved_rate`
- `top1_topic_hit_rate`
- `topic_hit_rate`
- `misses`
- `canonical_expected`
- `canonical_retrieved`

The evaluator uses bilingual topic aliases so Thai and English topic names can match the same canonical concept.

## What To Explain To A Mentor

The important distinction:

- Retrieval evaluation asks: “Did the system find the right material?”
- Answer evaluation asks: “Did the model use the material to produce a good answer?”
- Live query evaluation asks: “Is RAG better than a baseline LLM for this user query?”

If answer scores are bad, inspect in this order:

1. Retrieved chunks.
2. Language and answer-type preference.
3. RAG answer.
4. Baseline answer.
5. Judge rationale.
